# Docs Tutorial: General

这组教程用于指导如何在本地获取 `RMS-Docs`、修改 Markdown 文档，并将修改通过 PR 提交到仓库中。

考虑到需要此教程的，计算机水平都相对薄弱。此教程会倾向于傻瓜式教学。有一定计算机基础的，可根据自己实际水平，自行操作。

## 这组教程包含什么

本目录下共有三篇正式教程，建议按顺序阅读：

### 1. 环境准备与获取文档

文件：[1. Setup.md](/Docs%20Tutorial/1.%20Setup.md)

这一篇主要介绍：

- 下载并安装 VS Code
- 安装中文插件
- 下载并安装 Git
- 在 VS Code 中登录 GitHub
- 将 `RMS-Docs` 仓库拷贝到本地

### 2. 修改文档

文件：[2. Edit Docs.md](/Docs%20Tutorial/2.%20Edit%20Docs.md)

这一篇主要介绍：

- 修改前先确认本地 `master` 已经是最新的
- Markdown 基础语法
- Markdown 表格语法
- 表格的常见修改方式
- 如何借助网页 AI 辅助整理文档内容

### 3. 开分支并提交 PR

文件：[3. Submit PR.md](/Docs%20Tutorial/3.%20Submit%20PR.md)

这一篇主要介绍：

- 如何创建自己的分支
- 如何提交本地修改
- 如何推送到远端
- 如何创建 PR 并等待审核
- 如何在合并后清理本地分支

## 推荐阅读顺序

如果你是第一次接触这套流程，建议按下面顺序阅读：

```text
0. General -> 1. Setup -> 2. Edit Docs -> 3. Submit PR
```

这套顺序对应的实际操作逻辑是：

```text
先完成环境准备 -> 先同步最新的 master -> 再修改文档 -> 最后开分支、提交并发起 PR
```

如果你已经完成环境准备，只想学习怎么改文档，可以直接从 [2. Edit Docs.md](/Docs%20Tutorial/2.%20Edit%20Docs.md) 开始。
