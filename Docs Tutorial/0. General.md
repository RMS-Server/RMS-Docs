# Docs Tutorial: General

这组教程用于指导如何在 GitHub 上修改 `RMS-Docs` 文档，并通过 PR 提交修改。

考虑到需要此教程的，计算机水平都相对薄弱。此教程会倾向于傻瓜式教学。有一定计算机基础的，可根据自己实际水平，自行操作。

## 这组教程包含什么

本目录下共有三篇正式教程，建议按顺序阅读：

### 1. 环境准备与 Fork 仓库

文件：[1. Setup.md](/Docs%20Tutorial/1.%20Setup.md)

这一篇主要介绍：

- 注册 GitHub 账号（如果没有的话）
- Fork `RMS-Docs` 仓库到自己账号下

### 2. 修改文档

文件：[2. Edit Docs.md](/Docs%20Tutorial/2.%20Edit%20Docs.md)

这一篇主要介绍：

- 在 GitHub 网页上编辑文档
- Markdown 基础语法
- Markdown 表格语法
- 表格的常见修改方式

### 3. 提交 PR

文件：[3. Submit PR.md](/Docs%20Tutorial/3.%20Submit%20PR.md)

这一篇主要介绍：

- 如何提交修改并发起 PR
- 等待审核和合并
- 合并后同步自己的 Fork

## 推荐阅读顺序

如果你是第一次接触这套流程，建议按下面顺序阅读：

```text
0. General -> 1. Setup -> 2. Edit Docs -> 3. Submit PR
```

这套顺序对应的实际操作逻辑是：

```text
Fork 仓库（只需一次） -> 找到文件并编辑 -> 提交修改 -> 发起 PR -> 等待合并
```

如果你已经 Fork 过仓库，可以直接从 [2. Edit Docs.md](/Docs%20Tutorial/2.%20Edit%20Docs.md) 开始。

## 关于 Fork 工作流

本教程采用 **Fork 工作流**，这是一种常见的开源协作方式：

```text
原仓库 (RMS-Server/RMS-Docs)
    ↓ Fork（只需一次）
你的仓库 (你的用户名/RMS-Docs)
    ↓ 网页编辑 + 提交
你的仓库的分支
    ↓ Pull Request
原仓库
```

这种方式的优点：

- 不需要安装任何软件，完全在浏览器中完成
- 不需要原仓库的直接写入权限，任何人都可以贡献
- 你的修改会经过审核后才会合并到原仓库
