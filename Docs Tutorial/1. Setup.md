# Docs Tutorial: Setup

本教程用于指导如何注册 GitHub 账号并 Fork `RMS-Docs` 仓库。只需完成一次，后续修改文档时无需重复。

## 1. 注册 GitHub 账号

如果你已经有 GitHub 账号，可以跳过这一步。

### 操作步骤

1. 打开 GitHub 官网：[https://github.com](https://github.com)
2. 点击右上角 `Sign up` 按钮。
3. 按页面提示填写邮箱、密码、用户名。
4. 完成邮箱验证。
5. 登录你的 GitHub 账号。

### 说明

- 用户名建议使用容易记住的名字，后续提交记录会显示这个名字。
- 邮箱建议使用常用邮箱，方便接收通知。

---

## 2. Fork RMS-Docs 仓库

> [!IMPORTANT]
> 这一步非常重要。你需要先 Fork 仓库到自己账号下，才能在后续提交修改并发起 PR。

### 操作步骤

1. 登录你的 GitHub 账号。
2. 在浏览器中访问原仓库地址：[https://github.com/RMS-Server/RMS-Docs](https://github.com/RMS-Server/RMS-Docs)
3. 在仓库页面右上角找到 `Fork` 按钮，点击它。

![Fork 按钮位置](/Docs%20Tutorial/Assets/setup-fork-button.png)

图 1：在原仓库页面右上角找到 Fork 按钮。

4. 在弹出的页面中，确认 Owner 是你自己的账号，仓库名保持 `RMS-Docs` 不变。
5. 点击 `Create fork` 按钮。
6. 等待 Fork 完成，完成后会自动跳转到你自己账号下的 `RMS-Docs` 仓库页面。

![Fork 完成后的页面](/Docs%20Tutorial/Assets/setup-fork-complete.png)

图 2：Fork 完成后，你会在自己账号下看到一个 RMS-Docs 仓库。

### 说明

- Fork 的意思是在你自己账号下创建一个原仓库的副本。
- 你对这个副本有完全的写入权限，可以自由修改。
- 后续你会向原仓库提交 PR，而不是直接修改原仓库。
- Fork 只需要做一次，以后不需要重复 Fork。

---

## 3. 确认 Fork 成功

### 如何确认

Fork 成功后，你可以在自己的 GitHub 主页看到这个仓库：

1. 点击 GitHub 右上角的头像。
2. 选择 `Your repositories`。
3. 在列表中应该能看到 `RMS-Docs`。

或者直接访问：`https://github.com/你的用户名/RMS-Docs`

### 说明

- 以后修改文档时，都是访问你自己 Fork 的仓库，不是原仓库。
- 你自己的仓库地址和原仓库地址的区别在于用户名不同。
