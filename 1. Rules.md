# 服务器规定（暂行）

> *最终解释权归CXU管理层所有*

## 总纲

1. Conflux Union 同川世界 是一个致力探索、扩展原版科技树的技术生存服务器。
2. 服务器最高权力归服主所有。管理层在服务器运维、管理、发展等方面，协助服主。
3. 服务器将努力营造一个和谐、友好、团结的游戏氛围，一同进步。应有一个公平的游戏环境，尽可能使所有玩家都能拥有良好的游戏体验。
4. 服务器为原版技术生存服务器。在以生存服务器为中心的基础上，管理层将尽力维持服务器的技术向与原版性。
5. 服务器以探索、扩展原版科技树为目标。这里的“原版”泛指 Mojang 所有版本的任意特性。在此基础上，一切富含技术力的玩法都是接纳且鼓励的。

## CXU 管理制度规定

> **📌 简版说明**
>
> **管理员**：有心、有力、有责——愿意参与、有时间精力投入、承担管理责任。  
> **协管员**：有心、力有限、责有限——经筛选的协助角色，参与部分事务与重大投票、提供支持，但不承担完整管理职责。  
> **普通玩家**：不做额外要求，自由游玩。
>
> 协管员不是"活跃就能当"，需经管理层评估筛选。
> 协管员不参与日常管理，不进管理群，但在重大事务上有投票权（票权与管理员一致）。
> 对服务器事务长期消极、不愿贡献的协管员，将被取消资格。

---

### 一、制定依据

根据《总纲》及实际情况，经管理层讨论研究，制定此规定。

### 二、管理层组成与职责

1. 在服主之外，从活跃玩家中挑选心智年龄较高、责任心较强的，结合管理层意见与个人意愿，加入管理层。
    1. 新管理由老管理提名，经投票决定是否纳入管理层。纳新时可优先从协管员中考虑，但不以此为唯一途径。
2. 管理层内部地位平等，决策经投票表决。服主票权等效 1.5 票。赞成比例达到 2/3 及以上方可通过。
    1. 影响全服未来的重大事务（如换周目、合服等），要求最低参与投票人数 5 人（管理员与协管员合计）。
    2. 服内一般事务，由管理员投票决定，要求最低参与投票人数 3 人。
3. 管理层有参与决策的权利，也有对服务器进行维护、管理的责任。管理在活跃度上应起到带头作用。
4. 长期活跃情况低迷、未能承担起管理权责者，可经投票将其移出管理层。若因客观原因无法继续承担管理职责，但仍愿意参与部分服务器事务，经管理层评估后可转为协管员。待其时间充沛后，可优先再次选入管理层。
5. 管理层人员变更，适用于本规定 **二.2.1** 的重大事务相关规定。

### 三、协管员制度

#### 3.1 定位

协管员是独立于管理层的**协助与观察角色**，不属于管理层，不承担完整的管理职责，但作为经过筛选的、对服务器有贡献意愿的成员，在重大事务上享有投票权。

#### 3.2 来源

协管员从以下两类人员中筛选产生：

- **A 类（玩家晋升）**：在服时间较长、活跃度较高、有一定社区贡献的普通玩家，经管理层提名与评估后纳入。
- **B 类（管理转任）**：因现实原因无法继续担任管理员，但仍愿意参与部分服务器事务的前管理层成员，经评估后转任。

两类来源的协管员在权限和待遇上完全一致。前管理身份不自动等于协管员资格，仍需经评估确认。

#### 3.3 筛选标准

协管员的筛选由现任管理层集体评估，重点考察以下方面：

| 维度 | 说明 |
|---|---|
| 活跃度与在服时间 | 基本门槛，但不是唯一标准 |
| 规则意识 | 是否尊重服务器现有规则，有无违规记录 |
| 沟通与处事风格 | 是否理性，是否容易情绪化或引发社区矛盾 |
| 大局观 | 是否能从全服利弊出发思考问题 |
| 管理层整体评估 | 该人选加入后，是减轻还是增加管理负担 |

若管理层中有成员提出"该人选可能给管理工作带来额外负担"并给出合理理由，应优先搁置该提名。

#### 3.4 权限与职责

**拥有的权限：**

- 进入管理员及协管员专属频道，参与重大事务讨论
- 协助处理部分日常事务（如新玩家引导、轻微纠纷调解、活动协助等）
- 游戏内权限对标 helper（MCDR-2）
- 参与重大事务投票，票权与管理员一致

**不拥有的权限：**

- 不进入管理员专属群/频道
- 不拥有后台权限
- 不拥有生存服 OP 或 admin（MCDR-3）及以上权限
- 不参与一般事务投票（一般事务由管理员内部决定）

#### 3.5 退出机制

- 协管员若长期不活跃，管理层有权取消其协管员身份，回归普通玩家。
- 协管员若对服务器事务持续消极、不愿贡献，经管理层评估后可取消其协管员身份。
- 协管员若在任期间出现严重违规或给管理工作造成实质负担，管理层可投票移除。
- 协管员人员变更，适用于本规定 **二.2.2** 的一般事务相关规定。

### 四、权限

1. 服主应拥有服务器一切权限。不同管理根据能力分配相应的权限。

#### 4.1 后台权限

1. 除服主外，后台权限仅对管理层内、有能力且负责维护的玩家开放。
2. 应确保至少有两名活跃的管理拥有后台维护能力。

#### 4.2 权限等级划分

*说明：高等级拥有更低等级的所有权限。*

| 权限等级 | MCDR 等级 | 描述 | 权限说明 |
|---|---|---|---|
| guest | MCDR-0 | 默认权限。参观玩家、观察期玩家。 | 无特殊操作权限 |
| user | MCDR-1 | 普通玩家 | 可对所有子服创建备份；可同步镜像2；可启动镜像服、建筑服 |
| helper | MCDR-2 | 协管员 | 可对所有创造服、建筑服、镜像2回档；可删除备份 |
| admin | MCDR-3 | 管理层玩家 | 可对生存服、镜像1回档；可删除备份 |
| owner | MCDR-4 | MCSM | 等同于后台权限 |

**补充说明：生存服 OP**

1. 权限等级 `admin`（MCDR-3）及以上即可拥有。
2. 生存服始终开启 `op 禁止作弊`。请勿关闭此特性。

#### 4.3 QQ 群、KOOK 等社区管理

1. 以"谁负责谁管理，谁管理谁负责"的原则，拥有权限的也需要对此社区出现的问题负责。
2. 为确保消息处理及时，社区的管理员需要确保自己时间充裕，能相对及时地处理社区中的问题。
3. 公开社区里，管理员需要尤为注意自己的发言，避免负面影响。

#### 4.4 GitHub 文档编辑权限

- 管理层玩家根据需要向服主申请。

#### 4.5 权限滥用

管理或协管员若借助权限之便违反服规，从重处理。

## RMS审核制度规定及宣传相关流程

### 一、制定依据

- 根据《总纲》规定，结合管理层意见，基于实际情况制定此规定。  
- 该篇根据服内情况会较为频繁的调整，以确保审核难度、流程合适。

### 二、宣传职责

- 宣传由管理层负责，全服共同参与。
- 欢迎有志于此且有能力的玩家参与视频等宣传内容的制作。

### 三、宣传内容

1. 形式可多样，但考虑到平台及受众，现以视频为主。
2. 平台的选择。Bilibili 为传统的、主要的平台；抖音、YouTube 等以往未曾涉足的新平台也应给予足够的重视。条件允许的情况下，应针对不同平台推流的不同机制，对宣传内容进行适当调整。
3. 根据2024年暑假的会议讨论结果，制定了“两条腿走路”的宣传策略与路线。详情如下：
   1. 通过高技术力的内容，吸引技术实力较强的玩家。
   2. 通过叙事的、娱乐性较强的内容，吸引生电社区边缘玩家。
   - 理想情况下，应尽量使视频发在一个账号上，累积粉丝与流量。

### 四、审核负责人

- 审核交予管理层内可靠的管理长期负责。尽量避免经常更换负责人导致多次交接，以确保审核标准不变。
- 现审核负责人：重华、茜尔芙。


### 五、审核流程

#### 1. 一审

1. 请填写问卷（链接：https://survey.rms.net.cn/s/T8MSSeX4JGItZ8xq8fJlF）以便让我们更好地认识你。根据你填写的倾向方向，将会有不同的问卷内容。
2. 问卷填写过程中欢迎查阅 Wiki、进游戏尝试。但请不要求助于他人。
3. 我们欢迎任何人携带作品参与审核。但请准备好截图、投影或存档。若有提交的作品，部分问卷内容可选择性填写。
4. 若问卷通过，我们将尽量确保在问卷提交后的一周内通过问卷中留下的联系方式联系你进行二审。（因此，请务必注意，不要填错。否则无法与你取得联系。）


#### 2. 二审

1. 一审通过后，我们可能还有一些问题希望与你沟通、了解。如果你对我们服有疑问希望得到解答，也可在此阶段询问。
2. 如果问卷中填写的信息有误，可以在此阶段沟通修正。
3. 若二审通过，管理员将拉你进入 RMS 大群，并可获取白名。但请注意，审核仍未结束，请务必注意观察期。

#### 3. 观察期

1. 我们将在一段时间内观察新玩家活跃度情况、工程参与情况、与其他玩家的相处情况等，以确保新玩家能融入玩家群体。
2. 为帮助新玩家，将指派一名管理“一带一”进行帮扶。新玩家若有任何问题、任何情况，都可直接向管理反应。管理将尽己所能地提供帮助、协助解决。
3. 若新玩家因客观情况，一段时间内无法满足观察期的要求，可与管理说明情况，延后观察期。
4. 观察期结束后，将由管理层根据这段时间的情况进行投票，决定该名新玩家能否保留白名单。




## 服内玩家的家、机器、建筑建造相关规定

- 所有工程项目均需先**通知管理**进行选址评估，通过后方可开始建造。谨慎开坑，避免烂尾项目
  - 1.21.4 可优先联系**千万、茜尔芙**
  - 1.17.1 可优先联系**重华**

### 家
- 选址报备后请填写在线文档玩家的家坐标报备表

### 机器
- 同种机器请勿**重复建造**（升级迭代除外）
- 机器选址应在允许的范围内尽量便于玩家来往
- 玩家家中不得修建机器

### 建筑
- 所有公共建筑必须为服内成员原创

2. 打印机仅限建筑工程使用，由项目负责人负责监管：
   - 1.17.1：仅允许使用 [Litematica-Printer-Easyplace-Extension](https://github.com/aria1th/litematica-printer)
